# 设计稿导入改造功能记录

## 背景

此前 Web 端主要支持从空白参数开始分阶段生成剧本。对于已经有故事大纲、人物设定、分幕说明或线索设计的用户，缺少“先导入，再在编辑器中继续改造”的入口。

## 本次实现

- 新增 `/scripts/import` 页面，支持粘贴已有设计稿并填写剧本基础信息。
- 新增 `POST /api/scripts/import`，使用当前登录用户创建导入草稿。
- 新增 `ScriptDraftImportService`，将导入文本写入现有 `scripts / acts / scenes` 数据结构。
- 在 `/scripts` 作品列表页增加“导入设计”入口。
- 导入完成后跳转到 `/editor/[scriptId]`，复用现有编辑器保存、版本和后续改造能力。

## 当前边界

- 第一版只支持文本粘贴，不解析 docx、pdf 或图片。
- 第一版不调用 AI 做结构化抽取，避免引入额外计费和模型失败路径。
- 导入内容按空行拆成场景片段，统一归入“导入原稿”幕。
- 专家难度在当前 Supabase 类型只允许到 `advanced` 的环境中会落库为 `advanced`。

## 后续演进

- 增加 AI 结构化审阅：从原稿中抽取人物、线索、真相复盘和主持手册候选内容。
- 增加导入审阅页：左侧原文，右侧结构化结果，用户确认后再入库。
- 支持 docx/pdf 文本提取。
- 支持“基于导入稿补齐角色剧本 / 重构诡计 / 统一风格”等改造动作。
