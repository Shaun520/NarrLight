# NarrLight 开发指令

在本应用中进行代码开发、审查和重构时，使用 Codex skill：`karpathy-guidelines`。

## 代码质量规则

- 在进行非平凡修改前，先说明假设和可验证的成功标准。
- 优先使用能解决当前问题的最小实现。
- 保持改动精准：只修改当前任务直接需要的文件。
- 遵循项目现有的模式、命名、组件结构和样式习惯。
- 不引入未被要求的抽象、可选配置或无关清理。
- 只移除本次改动造成的未使用代码。
- 发现无关的废弃代码或风险时，只说明问题，不直接修改。

## 验证要求

根据改动范围选择最小但有意义的验证方式：

- 修改 TypeScript、React、Next.js 或共享库代码后，运行 `pnpm lint`。
- 修改行为逻辑或校验逻辑时，运行相关的 Vitest 测试。
- 修改路由、服务端/客户端边界、配置或生产行为时，运行 `pnpm build`。
- 如果某个命令无法运行，说明原因和剩余风险。

## 项目说明

- 本项目是 Next.js 应用，使用 React 19、TypeScript、Supabase、Ant Design、Zustand、Vitest 和 ESLint。
- UI 改动需保持与现有仪表盘和组件约定一致。
- 除非任务明确要求，否则保留现有中文产品文案和领域术语。
- 提交仓库格式，可参考：fix(editor): 编辑保存后清除脏快照防止内容回退（提交信息一定要是中文）
