# 快速验证指南：叙光平台

**日期**: 2026-06-17

本指南定义了 MVP 阶段核心功能的端到端验证场景，用于确认功能正确性。

## 前置条件

- Node.js 20+ 已安装
- Supabase 账号已注册（免费套餐即可）
- Supabase CLI 已安装（`npm install -g supabase`）
- AI Provider API Key 已配置（DeepSeek V4 Pro / GLM 5.1）
- 项目依赖已安装（`npm install`）

## 环境搭建

```bash
# 1. 克隆项目并安装依赖
git clone <repo-url> && cd NarrLight/narr-light
npm install

# 2. 配置环境变量
cp .env.example .env.local
# 编辑 .env.local，填入：
#   NEXT_PUBLIC_SUPABASE_URL=<your-supabase-url>
#   NEXT_PUBLIC_SUPABASE_ANON_KEY=<your-supabase-anon-key>
#   SUPABASE_SERVICE_ROLE_KEY=<your-supabase-service-role-key>
#   DEEPSEEK_API_KEY=<your-deepseek-api-key>
#   GLM_API_KEY=<your-glm-api-key>

# 3. 初始化 Supabase 数据库（本地开发）
npx supabase start
# 首次启动会自动执行 migrations/ 下的 SQL 脚本，创建表结构

# 4. 启动开发服务器
npm run dev
```

## 验证场景

### 场景 1：用户注册登录

**目标**：验证手机号+验证码登录流程

1. 访问登录页，输入手机号，点击「发送验证码」
2. 输入验证码，点击「登录」
3. **预期**：登录成功，跳转至剧本列表页，页面显示用户昵称和免费额度

### 场景 2：剧本 AI 全本生成

**目标**：验证 AI 剧本生成核心流程（用户故事 1）

1. 点击「新建剧本」，填写参数：题材=情感、6人、5小时、新手难度、古风背景、家国亲情立意
2. 点击「生成剧本」
3. **预期**：
   - 生成任务启动，页面显示流式进度
   - 生成完成后，剧本包含：人物剧本（6个角色）、组织者手册、线索卡、真相复盘
   - 人物情感线完整，符合古风背景
4. 选中第二幕，请求「增加公共搜证环节」
5. **预期**：仅第二幕新增线索与搜证流程，第一幕及后续幕不受影响
6. 打开版本历史面板
7. **预期**：显示初始生成版本和修改版本，支持一键回退

### 场景 3：时间线校验

**目标**：验证时间线冲突检测和可视化（用户故事 2）

1. 导入一部已知存在时间线冲突的测试剧本
2. 进入时间线校验模块，点击「执行校验」
3. **预期**：
   - 生成可视化时间轴，按角色分色显示
   - 冲突点红色标注，显示冲突描述（如「角色A在20:00同时出现在两地」）
4. 在时间轴上手动修正冲突节点
5. **预期**：修正后冲突消除，剧本对应原文同步更新

### 场景 4：逻辑闭环校验

**目标**：验证逻辑漏洞检测和自动修复（用户故事 2）

1. 对测试剧本执行「逻辑校验」
2. **预期**：
   - 输出漏洞列表，按严重程度分级（严重缺陷/局部警告/优化提示）
   - 点击漏洞可跳转至剧本编辑页对应位置
3. 选中一处漏洞，点击「一键按建议修复」
4. **预期**：系统自动修正对应段落，关联内容同步更新

### 场景 5：线索卡管理

**目标**：验证线索卡自动分类、编辑、导出（用户故事 3）

1. 进入已生成剧本的线索卡管理页
2. **预期**：所有线索按类型（物证/口供/深入/隐藏）、章节、地点自动分类
3. 标记一条线索为「干扰项」
4. **预期**：线索显示干扰项标记，逻辑校验中正确识别
5. 点击「导出线索卡」，选择 PDF 格式、A5 尺寸
6. **预期**：生成 PDF 文件，版式整齐，中文渲染正确

### 场景 6：人物关系可视化

**目标**：验证人物关系图谱生成和编辑（用户故事 4）

1. 进入已生成剧本的人物关系图页面
2. **预期**：
   - 自动生成可交互的关系图谱（力导向布局）
   - 明线/暗线关系区分显示
   - 支持拖拽节点、缩放画布
3. 双击一条关系线，修改关系标签
4. **预期**：关系更新后图谱实时刷新

### 场景 7：中断续传

**目标**：验证 AI 生成中断后可继续

1. 触发全本生成，等待部分内容生成后关闭页面
2. 重新进入该剧本编辑页
3. **预期**：已生成内容保留，显示「继续生成」按钮，点击后从中断处继续

### 场景 8：合规拦截

**目标**：验证违规内容拦截

1. 创建剧本时输入包含违规敏感内容的创作要求
2. 点击生成
3. **预期**：请求被拦截，显示合规提示，引导修改
